# Интеграция профиля пользователя с бэкендом

## Описание

Компонент `UserProfile.vue` был обновлен для загрузки информации о пользователе с бэкенда при заходе на страницу профиля.

## Функциональность

### Автоматическая загрузка профиля
- При заходе на страницу профиля автоматически загружается информация о пользователе
- Используется адрес кошелька из подключенного Phantom Wallet
- Отображаются состояния загрузки и ошибок

### API Endpoint
- **URL**: `http://localhost:3000/api/user/profile/{walletAddress}`
- **Method**: `GET`
- **Path Parameter**: `walletAddress` - адрес кошелька пользователя

### Маршрутизация
- **С адресом**: `/profile/{walletAddress}` - профиль конкретного пользователя
- **Без адреса**: `/profile` - профиль текущего подключенного пользователя

### Кликабельные адреса в трейдах
- **Трейды**: Адреса аккаунтов в таблице трейдов теперь кликабельны
- **Top Holders**: Адреса в списке топ-холдеров также кликабельны
- **Навигация**: Клик по адресу перенаправляет на `/profile/{walletAddress}`
- **Визуальная обратная связь**: Кликабельные адреса выделены цветом и имеют hover-эффекты

### Обработка ответов
- **Успех (200)**: Профиль пользователя загружен
- **Не найдено (404)**: Пользователь не найден
- **Ошибка валидации (400)**: Невалидный адрес кошелька
- **Ошибка сервера (500)**: Внутренняя ошибка сервера

## Состояния компонента

### Новые переменные
- `userProfile` - информация о пользователе из БД
- `isLoading` - статус загрузки
- `error` - сообщение об ошибке

### UI состояния
- **Загрузка**: Спиннер с текстом "Загрузка профиля..."
- **Ошибка**: Иконка ошибки с сообщением и кнопкой "Попробовать снова"
- **Успех**: Отображение профиля пользователя

## Отображаемые данные

### Основная информация
- **Username**: Имя пользователя из БД
- **Avatar**: Аватар пользователя (fallback на дефолтный)
- **Wallet Address**: Адрес кошелька (сокращенный формат)

### Статистика
- **Followers**: Количество подписчиков
- **Following**: Количество подписок
- **Created Coins**: Количество созданных токенов (пока 0)

### Интерактивные элементы
- **Copy Address**: Копирование адреса кошелька в буфер обмена
- **View on Solscan**: Ссылка на Solscan для просмотра адреса
- **Edit Button**: Кнопка редактирования (пока не функциональна)

## Функции

### `getWalletAddress()`
Получает адрес кошелька из параметров маршрута или подключенного Phantom Wallet

### `loadUserProfile()`
Загружает профиль пользователя с бэкенда

### `copyAddress()`
Копирует адрес кошелька в буфер обмена

### `formatWalletAddress(address)`
Форматирует адрес кошелька для отображения (сокращенный формат)

### `navigateToProfile(walletAddress)` (TokenChart.vue)
Перенаправляет на страницу профиля пользователя по адресу кошелька

## Логирование

Все операции логируются в консоль:
- `🔍 Загрузка профиля пользователя`
- `✅ Профиль пользователя загружен`
- `❌ Ошибка загрузки профиля`
- `✅ Адрес кошелька скопирован в буфер обмена`

## Безопасность

- Проверка подключения Phantom Wallet
- Валидация адреса кошелька на сервере
- Обработка сетевых ошибок
- Fallback значения для отсутствующих данных

## Тестирование

1. Подключите Phantom Wallet
2. Перейдите на страницу профиля через навигацию (автоматически сгенерируется URL с адресом)
3. Проверьте загрузку данных в консоли
4. Убедитесь что отображается корректная информация
5. Проверьте копирование адреса кошелька
6. Проверьте ссылку на Solscan
7. Попробуйте перейти на `/profile` без адреса (должен загрузиться профиль текущего пользователя)
8. **Новое**: Перейдите на страницу токена и кликните на адрес в трейдах
9. **Новое**: Кликните на адрес в секции Top Holders

## Обработка ошибок

### Типичные ошибки
- **Phantom Wallet не подключен**: Показывается сообщение о необходимости подключения
- **Пользователь не найден**: Ошибка 404 с возможностью повтора
- **Невалидный адрес**: Ошибка валидации
- **Сетевые ошибки**: Обработка с возможностью повтора
